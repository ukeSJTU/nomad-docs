### 1. 概述 (Overview)

本页面是项目的“UI 细节与文案风格指南”，集中定义了所有面向用户的表单校验规则、错误提示、确认信息和其他系统消息。其目的是确保整个应用的用户体验和“语调”保持高度一致、专业和清晰。

### 2. 全局消息与文案 (Global Messages & Copy)

此处定义适用于整个应用的通用消息。

#### 2.1 通用反馈提示 (General Feedback Toasts)

| 场景 (Scenario) | 类型 (Type) | 消息文本 (Message Text)                |
| :-------------- | :---------- | :------------------------------------- |
| 通用网络错误    | `Error`     | `网络连接失败，请检查您的网络后重试。` |
| 通用服务端错误  | `Error`     | `服务器开小差了，请稍后再试。`         |
| 通用成功操作    | `Success`   | `操作成功`                             |

### 3. 按功能模块划分的 UI 细节

此处按功能模块组织具体的校验规则和文案。

#### 3.1 用户模块 (User Module)

**3.1.1 用户注册 (User Registration)**

-   **表单字段校验规则:**
    | 字段 (Field) | 校验规则 (Validation Rule) | 错误提示文本 (Error Message) |
    | :--- | :--- | :--- |
    | 手机号 | 必填; 11 位数字; ... | `请输入正确的手机号码` |
    | 验证码 | 必填; 6 位数字 | `验证码格式错误` |
    | 密码 | 必填; 8-20 位; 包含字母和数字 | `密码长度需为8-20位，建议使用字母、数字和符号组合` |

**3.1.2 用户登录 (User Login)**

-   **表单字段校验规则:**
    | 字段 (Field) | 校验规则 (Validation Rule) | 错误提示文本 (Error Message) |
    | :--- | :--- | :--- |
    | 账户 | 必填; 手机号或邮箱格式 | `请输入正确的手机号/邮箱` |
    | 密码 | 必填 | `请输入密码` |

#### **3.2 机票模块 (Flight Module)**

**3.2.1 航班首页搜索 (Flight Homepage Search)**

-   **表单字段校验规则:**
    | 字段 (Field) | 校验规则 (Validation Rule) | 错误提示文本 (Error Message) |
    | :--- | :--- | :--- |
    | 出发地 | 必填 | `请选择出发城市` |
    | 目的地 | 必填 | `请选择到达城市` |
    | 出发日期 | 必填; 不可为过去日期 | `出发日期不能早于今天` |

**3.2.3 航班预订 (Flight Booking)**

-   **表单字段校验规则 (乘机人):**
    | 字段 (Field) | 校验规则 (Validation Rule) | 错误提示文本 (Error Message) |
    | :--- | :--- | :--- |
    | 姓名 | 必填; 必须与证件一致 | `请填写真实姓名` |
    | 身份证号| 必填; 必须符合格式 | `请输入正确的证件号码` |

#### 3.3 订单模块 (Order Management)

-   **二次确认弹窗:**
    | 操作场景 (Action Scenario) | 标题 (Title) | 内容/消息体 (Content/Body) | 确认按钮 | 取消按钮 |
    | :--- | :--- | :--- | :--- | :--- |
    | 取消未支付订单 | `确认取消订单` | `您确定要取消这个订单吗？取消后需要重新预订。` | `确认取消`| `返回` |
