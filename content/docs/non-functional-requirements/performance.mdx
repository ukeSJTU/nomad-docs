#### 1. 功能概述 (Feature Overview)

**本页定义了系统在正常负载和峰值负载下的关键性能指标，确保为用户提供流畅、不卡顿的访问体验。**

#### 2. 性能指标 (Performance Metrics)

| 项目             | 需求说明 (学期项目目标)                                                                                | 优先级          |
| :--------------- | :----------------------------------------------------------------------------------------------------- | :-------------- |
| **页面加载时间** | - 核心页面（首页、结果页、订单页）的首次内容绘制 (FCP) 应在 **1.5 秒** 内完成。                        | **Must Have**   |
| **API 响应时间** | - 核心 API（航班搜索、订单提交）的平均响应时间应在 **2 秒** 内。                                       | **Must Have**   |
| **并发能力**     | - **理论目标：** 系统应能支持 **50 个** 并发虚拟用户的访问而不出现明显性能下降或错误。                 | **Should Have** |
| **容量**         | - **理论目标：** 数据库设计应能支撑 **10,000 名** 用户的规模，且订单查询性能不随数据量增长而显著恶化。 | **Should Have** |
